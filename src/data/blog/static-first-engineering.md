个人博客很容易在技术选择上变重。评论、搜索、动画、主题、统计、订阅、内容组件都可以继续叠加，但叠加不应该破坏最基本的事情：页面要快，内容要可索引，部署要稳定。

未眠纸录的工程底线是 static-first。Astro 负责构建页面、组织内容集合、生成 RSS 与 sitemap；React 只用于确实需要交互的小岛，例如阅读操作和后续可能加入的局部控件。

## 为什么这样做

静态页面的优势直接而明确：首屏负担低，缓存友好，托管简单，失败面窄。GitHub Pages 足够承载这个站点的当前目标，CI 只需要验证格式、lint 和 build。

这也让设计系统更容易维护。全局样式、字体角色和文章组件都在源码里清晰可见，不依赖运行时状态才能成立。

## 性能不是口号

性能好的站点不只是 Lighthouse 分数高。它还应该在内容增长后保持构建可靠，在网络一般的设备上保持阅读顺畅，在维护者离开几个月后仍然能通过一条命令复现。

所以这个项目把复现流程写进 README，把部署过程写进 GitHub Actions，把不该提交的生成物放进 `.gitignore`。这些细节不华丽，但它们决定一个博客能不能长期存在。
